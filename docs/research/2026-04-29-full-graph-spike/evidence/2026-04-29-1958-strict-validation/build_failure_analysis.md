## Root cause

The CUTLASS package install at `Dockerfile.gb10:88-91` succeeds (`Successfully installed nvidia-cutlass-dsl-4.4.2` at line 718 of build_failure.log). The warmup step at L137 fails because the build container does not have access to the host CUDA driver library:

```
[disk_cache.py:778] CuTe DSL disk cache patch failed (CUTLASS not available):
  libcuda.so.1: cannot open shared object file: No such file or directory.
[kernel.py:37] CuTe DSL not available (CUTLASS not installed).
```

`docker build` does not pass `--gpus all` and there is no equivalent that exposes the NVIDIA driver libraries to the build context. CUTLASS *the python package* is present; CUTLASS *at runtime* needs `libcuda.so.1` from the host driver.

## Implications

1. The build-time warmup at `Dockerfile.gb10:137-144` has **always** failed silently — that's exactly what the `|| true` was hiding. The cache populated under `/opt/vllm/kernel_cache` in the image was empty in every nvllm:gb10 build to date.
2. The host bind-mount in `serve-cute-full.sh:82` (`$CUTE_COMPILE_HOST_CACHE_DIR:/opt/vllm/kernel_cache`) overlays the image-baked cache anyway, so even a populated image-cache wouldn't help warm serves. The cache that actually services warm serves is populated by the **first cold serve** (24s for the β-coop FULL key, per `evidence/2026-04-29-1634/precompile_run.log`).
3. The G1 detour's warmup.py prefill kwargs fix (also in `410d59390`) is correct and load-bearing — it matters at out-of-build precompile time and at first cold serve. Only the Dockerfile change in `410d59390` is wrong.
4. The G1 detour's `g1-v2-20260429-173404` evidence was generated by `cache_smoke.py` running in a freshly-launched container with `--gpus all` — that's why it succeeded. The build-time path was never tested before the Dockerfile change shipped.

## Fix

Restore `|| true` on `Dockerfile.gb10:137` (the warmup RUN). Either also restore it on the new `--verify-only` RUN at L141-144 or drop that step entirely — it can never pass without the previous step succeeding.
